# 执行历史

执行历史详细记录了面板中所有任务的运行状态、实时日志及耗时统计，是排查问题和监控任务运行的核心页面。

## 日志列表

每条日志记录包含以下信息：

| 字段 | 说明 |
| :--- | :--- |
| **日志 ID** | 唯一标识符 |
| **任务 ID** | 关联的任务标识 |
| **任务名称** | 关联的任务名称 |
| **任务类型** | 普通任务 (`task`) 或仓库任务 (`repo`) |
| **执行命令** | 实际执行的命令内容 |
| **状态** | 运行结果状态 |
| **耗时** | 从启动到退出的总耗时（毫秒） |
| **开始时间** | 任务开始执行的时间 |
| **结束时间** | 任务执行完成的时间 |

## 实时日志推流

即使任务仍在运行，也可以在日志详情页实时看到程序的控制台输出。

通过 WebSocket 连接实现实时推流：
- 任务运行中：连接到 `TinyLogManager`，订阅实时日志更新，并自动推送最近 100 行历史输出
- 任务已结束：直接从数据库读取已压缩存储的完整日志并返回
- 任务结束后自动推送最终完整日志内容

支持 ANSI 终端颜色代码的实时渲染，确保日志输出与终端一致。

## 状态说明

| 状态 | 说明 |
| :--- | :--- |
| `SUCCESS` | 任务成功完成，返回正常退出码 |
| `FAILURE` | 脚本运行时出错或程序异常终止 |
| `TIMEOUT` | 任务执行超出设定的最大运行时间，被系统强制中止 |
| `CANCELLED` | 任务被用户手动停止 |
| `RUNNING` | 任务正在执行中 |

## 搜索与过滤

支持以下维度的检索：
- **任务 ID**：精确匹配特定任务的日志
- **任务名称**：模糊匹配任务名称
- **状态筛选**：按 `SUCCESS`、`FAILURE`、`TIMEOUT`、`CANCELLED`、`RUNNING` 过滤

## 日志管理

### 清空日志

支持两种清空方式：
- **清空指定任务的日志**：传入 `task_id` 仅删除该任务的所有日志
- **清空全部日志**：不传 `task_id` 则删除所有任务的日志

### 删除日志

支持删除单条日志记录。

### 自动清理策略

在系统设置中可配置日志的自动清理策略，支持两种维度：
- **按天数清理**：保留最近 N 天的日志记录
- **按数量清理**：每个任务保留最近 N 条日志记录

可配置的清理维度包括：
- 任务日志保留天数 / 最大条数
- 系统通知保留天数 / 最大条数
- 推送日志保留天数 / 最大条数
- 登录日志保留天数 / 最大条数
- 调度日志保留天数 / 最大条数

## 日志存储

日志输出内容采用 gzip 压缩 + Base64 编码的方式存储在数据库中，有效节省存储空间。最大日志大小限制为 10MB，超出部分保留末尾内容。
