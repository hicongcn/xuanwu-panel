# 变量机密

变量机密提供了统一的环境变量管理功能，用于集中管理敏感数据和配置参数，保护机密信息的同时提高配置灵活性。

## 环境变量

### 基本属性

创建环境变量时可配置以下属性：

| 属性 | 说明 |
| :--- | :--- |
| **名称** | 变量名，建议使用大写字母加下划线，如 `DB_PASSWORD` |
| **值** | 变量值 |
| **备注** | 变量用途说明 |
| **类型** | 变量分类类型，默认 `normal` |
| **脱敏显示** | 控制在列表中是否隐藏值的显示 |
| **启用状态** | 控制该变量是否生效，禁用后值为空注入 |

### 全局注入

环境变量配置后，会全局注入到所有任务进程的环境变量中。任务执行时，子进程可通过 `os.environ`（Python）或 `process.env`（Node.js）直接访问。

### 重名合并

当存在多个同名环境变量时，系统采用 `&` 拼接的方式合并值，例如：

```
COOKIE=abc123&def456
```

禁用状态的变量值会被替换为空字符串参与合并。

### 任务关联

每个环境变量可以关联到多个任务，通过任务配置中的 `envs` 字段（逗号分隔的 ID 列表）建立关联。

删除环境变量时：
- 若有关联任务，系统会提示并返回关联的任务列表
- 选择强制删除时，自动清理所有任务中的引用关系

## Secret 脱敏

对于标记为脱敏的变量（默认所有新建变量均为脱敏状态），面板在浏览列表中会自动遮掩显示：

- 3 个字符以内：显示 `***`
- 4~6 个字符：保留首尾，中间显示 `***`
- 6 个字符以上：保留前 2 和后 2 位，中间显示 `****`

例如：`ABCDEFGH` 显示为 `AB****GH`

编辑环境变量时，需要通过覆盖更新的方式修改值，无法查看原始值。

## 加密存储

所有环境变量的值在数据库中采用加密方式存储，确保存储层的物理安全。即使数据库文件被直接读取，也无法获取明文数据。

## 注入机制

任务执行前，系统根据任务关联的环境变量 ID 列表查询变量值，格式化为 `NAME=VALUE` 数组后注入到子进程的环境变量中。

支持的任务级环境变量配置：
- 通过任务编辑页面选择关联的环境变量
- 可开启"注入全部环境变量"选项，自动获取所有已启用的变量

## API 接口

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/v1/env` | GET | 分页获取环境变量列表（支持 name/type 筛选） |
| `/api/v1/env` | POST | 创建环境变量 |
| `/api/v1/env/all` | GET | 获取所有环境变量（不分页） |
| `/api/v1/env/:id` | GET | 获取环境变量详情 |
| `/api/v1/env/:id` | PUT | 更新环境变量 |
| `/api/v1/env/:id` | DELETE | 删除环境变量（支持 `force=true` 强制删除） |
| `/api/v1/env/:id/tasks` | GET | 获取关联的任务列表 |
